## Implementation instructions

1. **Database schema** — Add `BulkActionBatch` model (fields in story file).

2. **Tests first** — One test per AC:
   - AC1: Selecting campaigns renders bulk toolbar with available actions
   - AC2: Bulk pause → confirmation modal lists all selected campaigns
   - AC3: Confirm → parallel platform API calls, returns success/fail summary
   - AC4: One platform failure does not roll back others — partial success is valid
   - AC5: Budget bulk action → each campaign gets its own change log entry with shared batchId
   - AC6: Viewer role → checkboxes not rendered, 403 on bulk action endpoints

3. **Implementation** — Bulk actions fan out to `PlatformApiService` in parallel using `Promise.allSettled()` (not `Promise.all()` — one failure must not cancel others). Each result is logged individually to `campaignChangeLog` with a shared `batchId`.

4. **Bulk endpoint:**
   - `POST /api/campaigns/bulk-actions` → body: `{ action, campaignIds[], value? }`
   - Returns: `{ batchId, succeeded: [], failed: [] }`
