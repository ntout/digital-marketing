## Implementation instructions

1. **Database schema** — Add `AiInsight` model using the exact fields in CONTEXT.md.

2. **Tests first** — One test per AC:
   - AC1: After sync completes → new AiInsight records generated within 10 minutes
   - AC2: GET /api/ai/insights → returns insights with type, headline, explanation, triggerData, recommendedActionType
   - AC3: POST /api/ai/insights/:id/action → routes to approval queue (if permission='approval') or executes (if 'autonomous')
   - AC4: POST /api/ai/insights/:id/dismiss → status = 'dismissed', no longer returned in default list
   - AC5: No insights in 24hrs → empty state response includes lastAnalyzedAt timestamp
   - AC6: GET /api/ai/insights?type=opportunity → filters correctly

3. **Insight generation job** — Create `ai-insights.job.ts` triggered after each sync. Generate insights by running a set of analysis rules against `metricRecord`. Start with these rule types:
   - `high_cpc_trend` — CPC increased >20% over 7 days for any campaign
   - `top_roas_campaign` — Campaign ROAS in top 10% of workspace, hasn't had budget increase in 14 days
   - `zero_conversions` — Campaign has spend but 0 conversions for 5+ days
   - `cross_platform_gap` — One platform has >2x ROAS of the lowest platform

4. **Deduplication** — Before creating an insight, check if an identical insight (same workspaceId + same rule type + same campaignId) exists with status='active' created in the last 48 hours. If so, skip — do not create duplicates.

5. **Dashboard widget** — Surface the top 3 unread insights as a widget on the main dashboard (US-005). Implement as a separate small component that polls `GET /api/ai/insights?limit=3&status=active`.
