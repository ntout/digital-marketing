We are implementing **US-003: Connect marketing platforms via OAuth**.

Read the full story in `stories/US-003-platform-connections.md`.

## Instructions

1. **Database schema** — Add `PlatformConnection` model. Fields are in CONTEXT.md. Tokens must be encrypted at rest using AES-256-GCM before writing, decrypted on read.

2. **Tests first:**
   - AC1: GET /api/connections/:platform/connect → redirects to correct OAuth URL with correct scopes
   - AC2: GET /api/connections/:platform/callback → stores encrypted tokens, creates connection record
   - AC3: Token refresh job runs when token is within 5 min of expiry
   - AC4: Failed refresh → status set to 'reconnect_required', admin notified
   - AC5: GET /api/connections → returns list with name, status, last-synced
   - AC6: DELETE /api/connections/:id → tokens deleted, status set to 'disconnected'

3. **Implementation** — Build a `PlatformApiService` class (see CONTEXT.md rule #2). All platform API calls in this and future stories go through this class. It handles token refresh, exponential backoff, and error normalization.

4. **Encryption utility** — Create `packages/utils/src/encrypt.ts` with `encrypt(plaintext)` and `decrypt(ciphertext)` using AES-256-GCM. Key from `process.env.ENCRYPTION_KEY`.

## Supported platforms at launch
- `google_ads` — OAuth scopes: `https://www.googleapis.com/auth/adwords`
- `meta` — OAuth scopes: `ads_read`, `ads_management`
- `tiktok` — OAuth scopes: `advertiser.read`
- `linkedin` — OAuth scopes: `r_ads`, `r_ads_reporting`
- `google_analytics` — OAuth scopes: `https://www.googleapis.com/auth/analytics.readonly`
