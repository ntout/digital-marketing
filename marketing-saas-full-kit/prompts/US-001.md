We are implementing **US-001: User authentication & workspace setup**.

Read the full story in `stories/US-001-workspace-auth.md`.

## Instructions

This is Phase 1 — the root of the entire app. Nothing else can be built without this.

Implement the following in order:

1. **Database schema** — Create Prisma models for `User`, `Workspace`, and `WorkspaceMember` using the exact field names in CONTEXT.md.

2. **Tests first** — Write tests for each acceptance criterion before any implementation:
   - AC1: POST /api/auth/signup creates user, sends verification email, returns 201
   - AC2: GET /api/auth/verify?token=x activates account and redirects
   - AC3: POST /api/workspaces creates workspace and assigns owner role
   - AC4: POST /api/auth/login returns httpOnly JWT access + refresh tokens
   - AC5: POST /api/auth/refresh silently refreshes expired access token
   - AC6: POST /api/auth/logout invalidates both tokens server-side

3. **Implementation** — Build route handlers, middleware, and service layer to pass all tests.

4. **Auth middleware** — Create `apps/web/src/lib/auth.ts` with `requireAuth(req)` (returns user object, throws 401 if invalid) and `requireRole(user, roles)` (throws 403 if insufficient). These will be called at the top of every protected Route Handler.

## File structure to create

- Multi-workspace switching UI
