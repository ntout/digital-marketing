## Instructions

This is Phase 1 ‚Äî the root of the entire app. Nothing else can be built without this.

Implement the following in order:

1. **Install Auth0 SDK** ‚Äî Add `@auth0/nextjs-auth0` to `apps/web`.

2. **Database schema** ‚Äî Create Prisma models for `User`, `Workspace`, and `WorkspaceMember` using the exact field names in CONTEXT.md. Note: `User` has `auth0Id` (unique), not `passwordHash` or `emailVerified` ‚Äî Auth0 owns those.

3. **Auth0 route handler** ‚Äî Create the SDK catch-all route at `apps/web/src/app/api/auth/[auth0]/route.ts`:
   ```ts
   import { handleAuth } from '@auth0/nextjs-auth0'
   export const GET = handleAuth()
   ```
   This provides `/api/auth/login`, `/api/auth/logout`, `/api/auth/callback`, `/api/auth/me` automatically.

4. **Tests first** ‚Äî Write tests for each acceptance criterion before implementation:
   - AC1: `/api/auth/login` redirects to Auth0 (mock Auth0 SDK in tests)
   - AC2: Auth0 callback ‚Üí `UserService.upsertFromAuth0()` called, user record created/found
   - AC3: New user with no workspace ‚Üí redirected to `/onboarding`; POST `/api/v1/workspaces` creates workspace + owner member
   - AC4: Unauthenticated request to `/(app)/` route ‚Üí redirect to `/login`
   - AC5: `/api/auth/logout` terminates session
   - AC6: `requireAuth(req)` returns correct user shape; missing session throws 401

5. **`requireAuth` helper** ‚Äî Create `apps/web/src/lib/auth.ts`:
   ```ts
   import { getSession } from '@auth0/nextjs-auth0'
   
   export async function requireAuth(req?: Request): Promise<AppUser> {
     const session = await getSession()
     if (!session) throw new ApiError(401, 'UNAUTHORIZED')
     // Look up local user by auth0Id, attach workspaceId + role
     const user = await UserService.findByAuth0Id(session.user.sub)
     if (!user) throw new ApiError(401, 'UNAUTHORIZED')
     return user
   }
   
   export function requireRole(user: AppUser, roles: Role[]): void {
     if (!roles.includes(user.role)) throw new ApiError(403, 'FORBIDDEN')
   }
   ```

6. **First-login sync** ‚Äî Create `apps/web/src/lib/services/UserService.ts` with `upsertFromAuth0(auth0User)` ‚Äî creates or updates the local `user` record from the Auth0 profile on each login.

7. **Protected layout** ‚Äî In `apps/web/src/app/(app)/layout.tsx`, use `getSession()` server-side and redirect to `/login` if no session exists.

8. **Workspace creation** ‚Äî Build the `/onboarding` page and `POST /api/v1/workspaces` Route Handler. After workspace creation, redirect to `/dashboard`.

## Do not build
- Custom login/signup UI (Auth0 Universal Login handles this)
- Password reset or email verification (Auth0 handles this)
- SSO or social login configuration (done in Auth0 dashboard, not in code)
- Multi-workspace switching UI (US-002)
- Billing
