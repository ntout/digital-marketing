## Implementation instructions

Implement in this exact order:

1. **Root workspace setup** ‚Äî Create `package.json` (pnpm workspaces root, no `main` field) and `pnpm-workspace.yaml` declaring `apps/*` and `packages/*`.

2. **Next.js app** ‚Äî Scaffold `apps/web` using `create-next-app` with: TypeScript, Tailwind CSS, App Router, `src/` directory, no ESLint (configured at root). Then initialise shadcn/ui: run `npx shadcn@latest init`, choose New York style, neutral base colour. Install these components: `button input badge dialog dropdown-menu table skeleton sonner card separator avatar`.

3. **Prisma package** ‚Äî Scaffold `packages/db`: `package.json` (name `@mktg/db`), run `prisma init --datasource-provider postgresql`, create `packages/db/src/clients.ts` exporting `dbWrite` and `dbRead` as two `PrismaClient` instances pointing to `DATABASE_URL` and `DATABASE_READ_URL` respectively.

4. **Utils package** ‚Äî Scaffold `packages/utils` (name `@mktg/utils`) with four modules in `src/`:
   - `encrypt.ts` ‚Äî `encrypt(text: string): string` and `decrypt(text: string): string` using Node.js `crypto` AES-256-GCM. Key read from `process.env.ENCRYPTION_KEY`.
   - `logger.ts` ‚Äî Winston logger, JSON format, level from `process.env.LOG_LEVEL ?? 'info'`. Export as default `logger`.
   - `secrets.ts` ‚Äî AWS Secrets Manager wrapper. `getSecret(name: string): Promise<string>`. Cache results in a `Map` with 5-minute TTL using timestamps. Use `@aws-sdk/client-secrets-manager`.
   - `email.ts` ‚Äî Resend client wrapper. `sendEmail({ to, subject, html, text }): Promise<void>`. Stub that logs in development (`process.env.NODE_ENV !== 'production'`).

5. **Worker app** ‚Äî Scaffold `apps/worker` (name `@mktg/worker`): TypeScript Node.js app. `src/index.ts` starts a BullMQ Worker connected to `REDIS_URL`. Reference `@mktg/db` and `@mktg/utils` as workspace dependencies.

6. **Root scripts** ‚Äî Add to root `package.json`:
   - `"dev": "concurrently \"pnpm --filter @mktg/web dev\" \"pnpm --filter @mktg/worker dev\""` 
   - `"db:migrate": "pnpm --filter @mktg/db exec prisma migrate dev"`
   - `"test": "pnpm -r test"`

7. **Docker Compose** ‚Äî Create `docker-compose.yml` at root:
   - `postgres` service: image `postgres:15`, port `5432:5432`, env `POSTGRES_PASSWORD=postgres POSTGRES_DB=mktg_dev`
   - `redis` service: image `redis:7`, port `6379:6379`

8. **Environment files** ‚Äî Create `.env.example` for `apps/web` and `apps/worker` with all variable names from `architecture/AWS-INFRASTRUCTURE.md`. No values ‚Äî placeholders only (e.g. `DATABASE_URL=`).

9. **ESLint + Prettier** ‚Äî Configure at root: `.eslintrc.js` extending `next/core-web-vitals` for web and `@typescript-eslint/recommended` for worker. `.prettierrc` with `singleQuote: true, semi: false, tabWidth: 2`.

10. **Vitest** ‚Äî Add `vitest.config.ts` to `apps/web` and `apps/worker`. Configure with `environment: 'node'` and `globals: true`.
