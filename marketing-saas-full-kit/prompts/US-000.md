## Implementation instructions

Implement in this exact order:

1. **Git branch first** — Before creating any files, create and switch to a dedicated feature branch:
   - `git checkout -b feature/US-000-project-bootstrap`
   - Do not make US-000 changes directly on `main`

2. **Root workspace setup** — Create `package.json` (pnpm workspaces root, no `main` field) and `pnpm-workspace.yaml` declaring `apps/*` and `packages/*`.

3. **Next.js app** — Scaffold `apps/web` using `create-next-app` with: TypeScript, Tailwind CSS, App Router, `src/` directory, no ESLint (configured at root). Then initialise shadcn/ui: run `npx shadcn@latest init`, choose New York style, neutral base colour. Install these components: `button input badge dialog dropdown-menu table skeleton sonner card separator avatar`.

4. **Prisma package** — Scaffold `packages/db`: `package.json` (name `@mktg/db`), run `prisma init --datasource-provider postgresql`, create `packages/db/src/clients.ts` exporting `dbWrite` and `dbRead` as two `PrismaClient` instances pointing to `DATABASE_URL` and `DATABASE_READ_URL` respectively.

5. **Utils package** — Scaffold `packages/utils` (name `@mktg/utils`) with four modules in `src/`:
   - `encrypt.ts` — `encrypt(text: string): string` and `decrypt(text: string): string` using Node.js `crypto` AES-256-GCM. Key read from `process.env.ENCRYPTION_KEY`.
   - `logger.ts` — Winston logger, JSON format, level from `process.env.LOG_LEVEL ?? 'info'`. Export as default `logger`.
   - `secrets.ts` — AWS Secrets Manager wrapper. `getSecret(name: string): Promise<string>`. Cache results in a `Map` with 5-minute TTL using timestamps. Use `@aws-sdk/client-secrets-manager`.
   - `email.ts` — Resend client wrapper. `sendEmail({ to, subject, html, text }): Promise<void>`. Stub that logs in development (`process.env.NODE_ENV !== 'production'`).

6. **Worker app** — Scaffold `apps/worker` (name `@mktg/worker`): TypeScript Node.js app. `src/index.ts` starts a BullMQ Worker connected to `REDIS_URL`. Reference `@mktg/db` and `@mktg/utils` as workspace dependencies.

7. **Root scripts** — Add to root `package.json`:
   - `"dev": "concurrently \"pnpm --filter @mktg/web dev\" \"pnpm --filter @mktg/worker dev\""` 
   - `"db:migrate": "pnpm --filter @mktg/db exec prisma migrate dev"`
   - `"test": "pnpm -r test"`

8. **Docker Compose** — Create `docker-compose.yml` at root:
   - `postgres` service: image `postgres:15`, port `5432:5432`, env `POSTGRES_PASSWORD=postgres POSTGRES_DB=mktg_dev`
   - `redis` service: image `redis:7`, port `6379:6379`

9. **Environment files** — Create `.env.example` for `apps/web` and `apps/worker` with all variable names from `architecture/AWS-INFRASTRUCTURE.md`. No values — placeholders only (e.g. `DATABASE_URL=`).

10. **ESLint + Prettier** — Configure at root: `.eslintrc.js` extending `next/core-web-vitals` for web and `@typescript-eslint/recommended` for worker. `.prettierrc` with `singleQuote: true, semi: false, tabWidth: 2`.

11. **Vitest** — Add `vitest.config.ts` to `apps/web` and `apps/worker`. Configure with `environment: 'node'` and `globals: true`.
