## Implementation instructions

1. **Database schema** — Add `CreativeAlert` model (fields in story file). Also confirm `metricRecord` ingestion from US-004 includes ad-level CTR — if not, update the sync job and normalizers to include adId and CTR at the ad level.

2. **Tests first** — One test per AC:
   - AC1: Ad CTR declining 5+ consecutive days AND >30% below ad set avg → CreativeAlert created
   - AC2: Permission = 'suggest' → AiInsight created, no platform action
   - AC3: Replacement ad exists + permission = 'autonomous' → fatigued ad paused, replacement activated
   - AC4: No replacement ad exists + autonomous → only suggestion insight, no platform action
   - AC5: Swap executed → two change log entries (pause + activate) with shared batchId
   - AC6: Activity log filtered to AI actions → creative swap entries show before/after creative names, each independently revertable

3. **Fatigue detection algorithm:**
   - Query last 10 days of CTR per adId
   - Detect 5+ consecutive days of decline (each day lower than the day before)
   - Compare current CTR to ad set's 7-day average CTR
   - Flag if current CTR is more than 30% below ad set average
   - Only flag each ad once per 7-day window (check CreativeAlert for recent entries)

4. **Replacement selection** — When swapping: query AdCreative for inactive ads in the same adSetId, ordered by most recent createdAt. Activate the first one found.
