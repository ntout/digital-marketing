> Paste CONTEXT.md into this session first, then paste this prompt.

> AiActionDispatcher and AiGuardrails must be working.

---

Read the full story in `stories/US-016-ai-budget-optimization.md`.

## Implementation instructions

1. **Database schema** — Add `AiGoals` model (fields in story file).

2. **Tests first** — One test per AC:
   - AC1: 3-day avg ROAS > target * 1.2 → budget increase proposed/executed up to 20%
   - AC2: 3-day avg CPA > limit * 1.2 → budget decrease proposed/executed up to 20%
   - AC3: CPA > limit for 7 consecutive days → pause proposed/executed
   - AC4: Proposed change would exceed hard limit → change is capped, hard limit noted in aiReason
   - AC5: Permission = 'suggest' → aiInsight created, no queue item, no platform call
   - AC6: Kill switch active → job exits immediately, logs 'halted by kill switch'

3. **Implementation** — Create a BullMQ job `ai-budget-optimization.job.ts` that:
   1. Calls `AiGuardrails.check()` first — exit immediately if not allowed
   2. Queries `metricRecord` for 3-day and 7-day rolling averages per campaign
   3. Compares against `AiGoals` thresholds
   4. Calls `AiActionDispatcher.dispatch()` for each action — never calls PlatformApiService directly
   5. Runs after every sync cycle completes (triggered by sync job completion event)

4. **aiReason format** — Must be human-readable. Example:
   > "Meta campaign 'Summer Sale' had a 3-day average ROAS of 4.2, exceeding the target of 3.0 by 40%. Increasing daily budget by 15% ($120 → $138). Hard limit of $500/day is not exceeded."

## File structure to create
