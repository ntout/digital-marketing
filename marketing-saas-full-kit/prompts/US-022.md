## Why this story is critical
US-016 (AI budget optimization) reads `aiGoals` to determine ROAS targets and CPA limits. Without this story, US-016 has nothing to evaluate against and will skip all campaign analysis.

## Implementation instructions

1. **Database schema** — Add `AiGoals` model using the exact fields in CONTEXT.md. Create a default record (roasTarget: 0, cpaLimit: null, currency: 'USD', evaluationWindowDays: 3) when a workspace is first created — add this to the workspace creation flow in US-001's workspace creation route/service.

2. **Tests first** — One test per AC:
   - AC1: GET /api/v1/settings/ai-goals → returns current goals or defaults
   - AC2: PATCH /api/v1/settings/ai-goals → validates and saves; returns 400 for invalid values
   - AC3: No goals configured (roasTarget = 0) → US-016 job receives null goals and logs skip message
   - AC4: Goals updated → next job run reads new values from DB (no caching)
   - AC5: Manager role → GET succeeds, PATCH returns 403
   - AC6: Currency dropdown shows ISO 4217 codes; selected currency persists

3. **Extend AI settings page** — Add a "Performance Goals" card to `apps/web/src/app/(app)/settings/ai-agent/page.tsx` using React Hook Form + Zod. Form fields: ROAS Target (number), CPA Limit (number, optional), Evaluation Window (number, 1–30), Currency (select). Disable all inputs if `user.role === 'manager'`.
