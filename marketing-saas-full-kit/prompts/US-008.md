## Implementation instructions

1. **Tests first** — Write a test for every acceptance criterion (AC1–AC6) in the story file before writing any implementation code.

2. **Implementation** — Build route handlers, service layer, and frontend components to pass all tests.

3. **Key details** — Anomaly detection runs as a BullMQ job after each sync. Use rolling 14-day average + 2 standard deviations. Create an AnomalyAlert model in Prisma. Notification bell reads unviewed alert count via a separate lightweight endpoint.

## File structure to create
```
packages/db/prisma/schema.prisma              (add AnomalyAlert model)
apps/worker/src/jobs/anomaly-detection.job.ts
apps/worker/src/services/AnomalyDetectionService.ts
apps/web/src/app/api/v1/alerts/route.ts
apps/web/src/app/api/v1/alerts/[id]/mute/route.ts
apps/web/src/__tests__/anomaly-detection.test.ts
```

## Do not build
- SMS/Slack notifications
- Custom alert formulas
- ML-based anomaly models
