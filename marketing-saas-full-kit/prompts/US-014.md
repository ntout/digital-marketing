## Why this story is critical
`AiGuardrails` is imported by every AI agent job (US-016, US-017, US-018, US-019). The kill switch and permission level checks must work correctly before any autonomous action is ever executed.

## Implementation instructions

1. **Database schema** — Add `AiWorkspaceSettings` model using the exact fields in CONTEXT.md. Create with defaults (all permissions = 'suggest', kill switch = false) when a workspace is first created.

2. **Tests first** — One test per AC:
   - AC1: GET /api/settings/ai-agent → returns settings with all action types and current permission levels
   - AC2: PATCH /api/settings/ai-agent → saves new permission levels, returns updated settings
   - AC3: AiGuardrails.check() → returns { allowed: false, reason: 'hard_limit' } when proposed change exceeds maxDailyBudgetUsd
   - AC4: AiGuardrails.check() → returns { allowed: false, reason: 'quiet_hours' } when current time is within quiet hours
   - AC5: Manager/Marketer/Viewer role → settings page loads but all inputs are disabled; PATCH returns 403
   - AC6: POST /api/settings/ai-agent/kill-switch → sets killSwitchActive = true, all AI jobs skip execution

3. **Fully implement AiGuardrails** — Replace the stub from US-013 with real logic:
```ts
AiGuardrails.check(workspaceId, actionType, proposedChange) => Promise<{
  allowed: boolean
  permissionLevel: 'suggest' | 'approval' | 'autonomous' | null
  reason: 'kill_switch' | 'quiet_hours' | 'hard_limit' | 'not_allowed_platform' | null
}>
```
This is the single function called at the top of every AI job before any action.
