## Implementation instructions

1. **Database schema** — Add `AdCreative` model (fields in story file).

2. **Tests first** — One test per AC:
   - AC1: GET /api/ads/:id/edit → returns all copy fields with current values and char limits
   - AC2: Field exceeding platform char limit → validation error before preview
   - AC3: Preview renders platform-accurate mockup data (not a live render — return structured data the frontend uses to render a mockup component)
   - AC4: Confirm → platform API call, change log entry created
   - AC5: Successful confirm → adCreative.locallyModified = true, fields updated locally without waiting for next sync
   - AC6: Cancel → no API call, no state change

3. **Character limits by platform** — Encode these as constants in `apps/web/src/lib/constants/ad-copy-limits.ts`:
   - Meta: headline 40 chars, primary text 125 chars, description 30 chars
   - Google Ads: headline 30 chars (×15), description 90 chars (×4)
   - TikTok: caption 150 chars
   - LinkedIn: headline 70 chars, description 70 chars

4. **Ad preview** — The preview endpoint returns structured data (not HTML). The frontend renders the mockup using a `<AdPreview platform="meta" fields={...} />` component.
