# Codex prompt — US-017: AI agent — cross-channel budget reallocation

> Paste CONTEXT.md into this session first, then paste this prompt.
> US-016 (single-platform budget optimization) must be complete.

---

Read the full story in `stories/US-017-ai-cross-channel-reallocation.md`.

## Implementation instructions

1. **Database schema** — Add `CrossChannelBudgetPool` model (fields in story file).

2. **Tests first** — One test per AC:
   - AC1: Pool configured → AI ranks platforms by 7-day ROAS, proposes shifting ≤15% from lowest to highest performer
   - AC2: Destination platform has no headroom → skip that platform, note in reasoning
   - AC3: Reallocation executed → two change log entries (decrease + increase) with shared batchId
   - AC4: Platform with sync error → excluded from evaluation, flagged in aiReason
   - AC5: Permission = 'approval' → single queue item describing the full cross-platform move
   - AC6: Any individual campaign would exceed maxDailyBudgetUsd → reallocation capped

3. **Implementation** — Extend the budget optimization job or create a separate `ai-cross-channel-reallocation.job.ts`. Key difference from US-016: this operates across platforms, so the decrease and increase are on different platform connections and must be dispatched as a batch (shared batchId).

4. **Headroom calculation** — Before proposing a budget increase on a destination platform, check: `currentBudget + proposedIncrease <= maxDailyBudgetUsd (from hard limits)`. If not, reduce the proposed increase to fit, or skip if no room at all.

## File structure to create
```
src/
  prisma/schema.prisma                      (add CrossChannelBudgetPool model)
  src/jobs/ai-cross-channel-reallocation.job.ts
  src/services/CrossChannelAnalysisService.ts
  tests/ai-cross-channel-reallocation.test.ts
```

## Do not build
- Currency conversion for multi-currency workspaces
- Reallocating between campaigns within the same platform (that's US-016)
