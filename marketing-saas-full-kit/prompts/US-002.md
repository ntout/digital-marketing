We are implementing **US-002: Team invitations & role management**.

Read the full story in `stories/US-002-team-roles.md`.

## Instructions

1. **Database schema** — Add `Invitation` model to Prisma schema (fields in story file).

2. **Tests first** — One test per acceptance criterion:
   - AC1: Admin POSTs to /api/invitations → email sent, pending record created
   - AC2: GET /api/invitations/accept?token=x → user added to workspace with correct role
   - AC3: GET /api/team → returns members with name, email, role, status
   - AC4: PATCH /api/team/:userId/role → role updated, effective immediately
   - AC5: DELETE /api/team/:userId → member removed, access revoked
   - AC6: Viewer attempting GET /api/team/settings → 403

3. **Implementation** — Routes, service layer, and role-checking middleware.

4. **Role middleware** — Create `requireRole(...roles)` middleware that wraps `requireAuth` and checks `req.user.role` against an allowlist. This is used by every protected endpoint going forward.
